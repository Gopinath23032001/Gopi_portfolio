India Power Outage Pattern & Grid Reliability Tracker

Python PostgreSQL Supabase Streamlit Plotly Power BI Open-Meteo API pandas psycopg2

The Problem

India's electricity distribution companies (discoms) do not publish outage data publicly. Residents, businesses, and researchers have no systematic way to understand which states are chronically unreliable, what conditions drive grid stress, or how to plan around outages. People are effectively flying blind.

This project builds that visibility layer using publicly available central-level data from the Central Electricity Authority (CEA), enriched with real-time weather observations to produce a composite risk score for each state.

Architecture

The pipeline combines two data sources: CEA's Daily Maintenance Report (Sub-Report 10), downloaded manually from npp.gov.in, and daily weather data fetched automatically from the Open-Meteo Historical API. Both flow into a PostgreSQL database hosted on Supabase, where they're joined, scored, and served to the Streamlit dashboard.

  • parse_report10.py — extracts unit-level outage records from raw XLS files into a clean DataFrame, including reason-text categorisation (Mechanical / Electrical / Fuel / Grid-Other / Unknown)
  • load_report10.py — upserts the DataFrame into PostgreSQL with idempotent conflict handling so the same file can be reloaded without creating duplicates
  • batch_load.py — runs the loader across an entire folder of files in date order, with per-file error handling and a summary report
  • fetch_weather.py — pulls daily max temperature and rainfall for all 36 states/UTs from the Open-Meteo API and loads them into the weather fact table
  • risk_model.py — computes the composite risk score and writes results to fact_risk_scores

Database Schema

A star schema with two dimension tables and three fact tables, hosted on Supabase (PostgreSQL 16). Dimensions hold stable descriptive data; facts hold time-series measurements joined by integer foreign keys.

  • dim_state — 36 states/UTs with region classification
  • dim_station — 82 power stations, auto-populated from report data
  • fact_outage_event — unit-level daily outage records (station, MW, category, reason text, timestamps)
  • fact_weather_event — daily max temperature and rainfall per state
  • fact_risk_scores — composite risk score per state per day with component breakdown

Key Finding — Chronic Outage Entries

During model development, I discovered that CEA's daily report tracks all currently open outages — not just new ones. Units that have been offline for years (some since 2004) appear in every daily report with identical MW values. This "chronic outage stock" accounts for 85%+ of entries.

This made supervised ML targets degenerate — binary "any outage today" gave a 100% positive rate; "above state median" gave 0%. Rather than hiding this, I documented it as a real data-structure finding and redesigned the risk model to a rule-based scoring approach, which is standard in operational risk contexts when supervised learning cannot be applied. The finding is documented openly in the dashboard's Methodology page.

Risk Scoring Model

The composite risk score combines three normalised components weighted by operational significance:

  • Outage Load (50%) — forced MW as a percentage of total MW under maintenance, normalised 0–1 across all state-days
  • Heat Stress (30%) — daily max temperature, capped at 45°C and normalised — captures cooling-system failures in thermal plants and demand spikes
  • Rain Stress (20%) — daily rainfall in mm, capped at 100 mm and normalised — captures equipment faults and transmission disruptions from heavy weather

States scoring above 0.66 are labelled High, 0.33–0.66 Medium, and below 0.33 Low. On the latest report date, Uttar Pradesh scored 0.720 (High) with 100% forced-outage load at 39.5°C, while Jharkhand scored 0.156 (Low) despite moderate forced load, reflecting cooler temperatures and higher rainfall.

SQL Analysis

Five core queries drive the analysis, using window functions and PostgreSQL's built-in corr() aggregate:

  • Worst-state forced-outage ranking by event count and total MW lost
  • Daily trend of forced MW across all states over time
  • Pearson correlation between max temperature and forced outage events
  • Hot-day vs. normal-day comparison (≥40°C vs. below) for average forced events
  • Root-cause breakdown: Mechanical (38%) · Grid/Other (26%) · Fuel (15%) · Electrical (6%) · Unknown (16%)

Streamlit Dashboard

A 4-page interactive dashboard with a live Supabase connection, built with Streamlit and Plotly:

  • National Overview — India choropleth map coloured by risk score, KPI cards, and all-India forced MW trend line
  • State Drill-Down — per-state risk score trend, reason-category donut chart, and temperature/rainfall overlay timeline
  • Risk Scoreboard — all states ranked by composite score with a colour-coded heatmap across all loaded dates
  • Methodology — data sources, risk formula, the chronic-outage finding, and honest caveats — documenting limitations openly rather than hiding them

Honest Caveats

This tool covers inter-state and large-capacity thermal/nuclear units monitored by CEA. Local distribution-level outages are not included as no public API exists. Weather data uses each state capital as a proxy — sub-state variation is not captured. Risk scores are normalised within the loaded date range, so a longer history would produce more stable results.
